# CiTT 精简说明

CiTT, also known as VeriCircuit Tutor, 是一个面向本科电路分析和生物医学仪器教学的仿真验证型 AI 导师。它的核心原则是：LLM 不负责给出数值真相，数值答案必须来自确定性的电路求解器和验证器。自然语言或图像输入可以被转换成结构化电路表示，导师层可以解释结果，但不能凭语言模型直接生成最终电压、电流、功率或单位结论。

## 项目定位

CiTT 不是通用 SPICE 替代品，也不是生物医学设备验证工具或临床决策系统。它是一个受控的教学环境：系统只在明确支持的电路范围内解析、验证、求解、检查并解释问题。

项目目标是提供可信的电路教学体验。学生可以得到解释，教师可以检查数值从哪里来；系统遇到不支持或含糊的问题时，应明确拒绝或标记不确定，而不是生成看似流畅但不可验证的答案。

## 核心流程

系统遵循以下权威链路：

```text
problem input -> Circuit IR -> validation -> solver -> verifier -> Solution Packet -> tutor output
```

Circuit IR 记录电路图、元件、分析模式、目标、假设、歧义和不支持特性。求解器产生数值结果，验证器检查内部一致性，导师层再基于已验证字段生成解释、视觉课程、推理提示或教师汇总。

## 主要能力

CiTT 当前包含五类核心能力：

- 求解器验证答案：支持范围内的电路由确定性引擎求解，并返回请求答案、来源信息和验证状态。
- 引导式视觉课程：通过已求解的数据包生成学习目标、公式步骤、图像焦点、常见错误、检查项和限制说明。
- 推理教练：学生先提交部分思路，系统给出局部提示，并在满足条件前延迟最终数值揭示。
- 生物医学仪器教学层：把已验证电路结果连接到信号链角色、安全提醒、常见实验错误、非理想因素、采样观察、CMRR 讨论和初步噪声估计等教学语境。
- 诚实边界处理：不支持或含糊的问题会被标记为 unsupported 或 ambiguous，而不是强行编造答案。

## 架构概览

后端是基于 FastAPI 的服务，围绕类型化模型和确定性服务模块组织。

- Parser service：把自然语言或图像输入转换为 Circuit IR。Gemini 可以作为可选解析器，但不能生成最终数值答案。
- Circuit IR models：定义支持的元件、节点、目标、假设、歧义、不支持特性、分析设置和生物医学元数据。
- Validator：在求解前检查拓扑、单位、元件值、目标、接地连通性、分析约束和功能边界。
- Solvers：使用 Modified Nodal Analysis 及相关确定性方法处理支持的 DC、AC、瞬态、二极管、运放、非理想运放、噪声和增量更新路径。
- Verifier：检查 KCL 残差、有限数值、适用的有符号功率平衡、AC 一致性和目标答案覆盖。
- Explainer and lesson builder：只从已验证数据包或确定性教学观察中生成教学内容。
- Reasoning coach：检查学生的思路框架，识别可能误解，返回诊断图和一个局部提示，并按条件控制最终答案揭示。
- Biomedical context layer：通过显式模板或有限拓扑特征检测附加教育性的生物医学意义。
- Visual layer：生成语义化电路布局数据，并把公开 schematic SVG 渲染交给 OptCPV。
- Frontend：React 工作区，负责导师 UI、API 编排、视觉焦点、课程步骤、聊天/教练状态和电路交互。

## 支持范围

当前支持的功能面包括：

- 电阻、独立电压源、独立电流源、接地、节点电压、元件电压、元件电流和元件功率。
- DC 工作点中把电容视为开路。
- 线性 R/C/L/source 电路的数值瞬态分析。
- 线性 R/L/C/source 电路的单频 AC 相量分析和 AC sweep。
- 电感在 DC 稳态、AC、sweep 和线性瞬态模式中的分析。
- Shockley 二极管 DC 工作点分析，使用 Newton-Raphson nonlinear MNA。
- 理想闭环运放 DC 分析。
- 教学型非理想运放宏模型，包括有限增益、电源轨、输出摆幅、输入失调、偏置电流、输入电阻、输出电阻、补偿电容、电流限制检查、slew-rate 提示、clipping recovery 提示和 AC gain-bandwidth 响应。
- 在标签和连接关系清晰时，通过 Gemini 把 schematic/image 解析成 Circuit IR。
- DC 电阻网络的 rank-1 增量更新路径。
- 配置化热噪声、shot noise 和 1/f 教学项的生物医学噪声传递估计。
- 用于透明性的 SPICE-like netlist 生成。
- 从 Circuit IR 通过 OptCPV 生成确定性 schematic。
- 答案来源信息，包括 parser、solver、矩阵、RHS 和 solution vector。

## 不支持范围

CiTT 会对当前边界外的问题显式拒绝或标记不支持。当前不支持：

- 任意波形瞬态源。
- 非线性瞬态分析。
- 晶体管和任意半导体模型。
- 任意受控源行为。
- SPICE 级厂商运放宏模型仿真。
- 对不可读、裁切、歧义或视觉不清晰 schematic 的保证识别。
- 生物医学安全认证、IEC 合规分析、漏电流设计验证、患者特异性决策支持、诊断、治疗或受监管医疗器械审查。

## 验证与可信模型

CiTT 的验证状态表示求解结果是否通过了当前支持范围内的可用检查。通过验证表示数据包满足验证、数值求解、KCL、适用的功率或 AC 一致性、单位处理和目标答案覆盖。它不代表电路达到完整 SPICE 精度，也不代表任何生物医学认证。

系统使用统一符号约定：

- 元件电压为 `V(nodes[0]) - V(nodes[1])`。
- 元件电流正方向为从 `nodes[0]` 到 `nodes[1]`。
- 有符号功率为电压乘以电流。
- 对源而言，负的有符号功率表示该源向电路供能。

## 教学设计

CiTT 试图让学生保留推理主动权，而不是一开始就得到包装好的最终答案。课程和教练层强调局部检查、表示方式切换、误解识别、视觉焦点、已验证引用和明确限制。

在生物医学仪器方向，系统会把电路结果连接到信号链和实验语境，但边界很清楚：这些语境用于教学，不用于设备批准、病人连接场景或临床声明。

## 关键差异

- 把语言表达能力和数值权威分开。
- 暴露答案背后的结构化电路表示和求解来源。
- 对不支持或含糊问题诚实拒绝，而不是猜测。
- 教学内容基于已验证数据包，而不是自由生成。
- 支持先提示、后揭示答案的推理教练模式。
- 加入生物医学仪器教学语境，同时保持安全和适用范围边界。

## 当前限制与风险

- 覆盖范围相对完整 SPICE 有意收窄。
- Gemini 解析是可选路径，输入无法安全结构化时应返回歧义。
- 生物医学语境依赖模板或有限拓扑检测，不是开放式生理推断。
- 学生画像和教师汇总目前是无持久化的 MVP payload。
- OptCPV 是 schematic 渲染真值源，因此图形输出依赖该集成。
- 评估结果受现有测试和 benchmark 范围限制，不能视为独立认证。
